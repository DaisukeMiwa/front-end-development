総合 Web コーディング入門ガイド

HTML・CSS・Javascript・jQuery を 1 ページで通読できる総合教材です。 まず 4 つの違いを押さえ、上から順に進めば、Web ページを「組む → 見せる → 動かす → 短く書く」までを体験できます。

00Web コーディング 4 つの役割

Web ページは 「組む（HTML）→ 見せる（CSS）→ 動かす（Javascript）→ 短く書く（jQuery）」 の 4 層で出来ています。まずはそれぞれの担当を押さえましょう。

HTML

ページの骨組み・構造を作るマークアップ言語。見出し・段落・画像・リンクなど「何を置くか」を決める。

CSS

HTML の見た目・レイアウトを整えるスタイルシート。色・サイズ・配置・アニメーションを担当。

Javascript

Web ページに動きを与えるプログラミング言語。クリック反応・通信・計算などを担当。

jQuery

Javascript を短く書ける外部ライブラリ。スライダーや演出などの定番プラグインも豊富。

4 分野の役割マップ

観点 HTML CSS Javascript jQuery
担当 構造（骨組み） 見た目（装飾・配置） 動き（処理・通信） 動きを短く書く道具
正体 マークアップ言語 スタイル言語 プログラミング言語 JS のライブラリ
例えるなら 家の柱・壁 クロス・塗装 家電・スイッチ 電動工具セット
学ぶ順 ① 最初に学ぶ ② 続けて学ぶ ③ 動きを足す段で ④ JS の後がベスト

Javascript と jQuery にフォーカス

JS と jQuery は同じ「ブラウザを動かす」目的でも、役割と書き方が違います。後の章のために、結論を先に押さえておきましょう。

Javascript

Web ページを動かすプログラミング言語そのもの。ブラウザに最初から備わっている本体。

jQuery

その Javascript を短く書きやすくしてくれる外部ライブラリ。便利な道具箱のような存在。

4 つの観点で比較

観点 Javascript jQuery
正体 プログラミング言語 JavaScript で書かれたライブラリ
役割の抽象度 やや低レベル（細かく書く） 高レベル（短く書ける）
書き方 行数が多めで明示的 数行でまとまる
学習の必要度 必須（基礎知識として） 便利／案件次第で選ぶ

JavaScript の基本構文（DOM 操作）

document.querySelector('セレクタ').style.プロパティ = 値;
document：HTML 文書全体
querySelector：CSS セレクタで要素取得
'セレクタ' / 値：対象と設定値
style：取得要素のスタイルへアクセス
プロパティ：CSS プロパティ名（キャメルケース）

例：すべての <p> 要素の文字色を赤にする

document.querySelector('p').style.color = 'red';

jQuery の基本構文

$('セレクタ').css('プロパティ', '値');
$()：要素を指定（jQuery オブジェクト化）
'セレクタ' / '値'：対象と設定値
.css：CSS を設定／取得するメソッド
'プロパティ'：CSS プロパティ名（ケバブケースのまま OK）

例：すべての <p> 要素の文字色を赤にする

$('p').css('color', 'red');

「同じことを書く」と何が違う？

代表的な操作で、JavaScript と jQuery の書き方を並べて比較してみましょう。

① id で要素を取得・テキストを変える

// JavaScript
const el = document.getElementById('title');
el.textContent = 'こんにちは';
// jQuery
$('#title').text('こんにちは');

② スタイルを変更

// JavaScript
const el = document.querySelector('#title');
el.style.color = 'red';
el.style.fontSize = '18px';
// jQuery
$('#title').css({
  color: 'red',
  fontSize: '18px'
});

③ クリックイベント

// JavaScript
const btn = document.getElementById('btn');
btn.addEventListener('click', () => {
  alert('押された');
});
// jQuery
$('#btn').on('click', () => {
  alert('押された');
});

④ 要素の追加 / 削除

// JavaScript（追加）
const li = document.createElement('li');
li.textContent = '新規';
document.querySelector('ul').appendChild(li);

// 削除
document.querySelector('#x').remove();
// jQuery（追加）
$('ul').append('<li>新規</li>');

// 削除
$('#x').remove();
ポイント

最近のモダンブラウザは JavaScript（ES6+）だけでもかなり短く書けるようになっています。 ただし jQuery を使うとセレクタ・複数操作のチェーン・互換性がさらに楽になります。 プロジェクトに合わせて選びましょう。

HTH3HTML の基本構成・タグ・属性

VS Code で ! を入力 → Tab キーを押すと、ほぼ毎回書く定型のセット（HTML5 雛形）が自動で展開されます。

定型セット

<!DOCTYPE html>     // HTML5 の宣言
<html lang="ja">      // lang 属性で日本語
<head>
  <meta charset="UTF-8">                                 // 文字コード
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>サイトのタイトル</title>
  <meta name="description" content="">           // サイトの説明（SEO の中核）
  <link rel="icon" href="image/favicon.ico">      // ファビコン（任意）
  <link rel="stylesheet" href="css/style.css"> // CSS 読み込み
</head>
<body>
</body>
</html>
2026 年版の補足
  • <meta name="keywords"> は Google・Bing が 2009 年以降ランキング要素として無視しており、現代の SEO では効果ゼロ（むしろスパム判定リスクあり）。代わりに <title> と <meta name="description"> を最適化するのが現代の主流。
  • viewport に maximum-scale=1.0 や user-scalable=no を入れる古いテンプレもあるが、ピンチズーム禁止はアクセシビリティ違反になるので入れない。
  • <link rel="icon"> はファビコン。任意だがブラウザタブで識別性が上がる。

タグ・要素・属性

HTML は開始タグと終了タグでテキストを挟むのが基本。開始から終了までを「要素（element）」と呼びます。 

<p>こんにちは</p>     // <p> が開始、</p> が終了。全体で 1 要素

タグには付加情報を書く属性を持たせられます。属性は「属性名＝属性値」のセット。 

<p class="hello">こんにちは</p>
//   ↑属性名  ↑属性値

HTH4基本タグ（p / h / ul / img / a）

使用頻度の高いタグを 1 セクションでまとめて押さえます。

段落・改行・見出し

タグ 役割
<p>〜</p> 段落（paragraph）。ひとかたまりの文章を囲う
<br> 改行（break）。終了タグ無しで単独で使う
<h1>〜<h6> 見出し（heading）。h1 が最重要、h6 が最弱。順番を守る 
<h1>大見出し</h1>
<h2>中見出し</h2>
<p>これは段落です。<br>ここで改行されます。</p>

リスト（ul / ol）

<ul>（順序なし）と <ol>（順序あり）は、必ず <li> とセットで使います。 

// 順序なし（メニュー・写真など）
<ul>
  <li>りんご</li>
  <li>みかん</li>
</ul>
// 順序あり（手順・ランキング）
<ol>
  <li>準備する</li>
  <li>実行する</li>
</ol>

画像とリンク（img / a）

<img src="image/hero.jpg" alt="代替テキスト" width="800" height="450" loading="lazy">
// src=画像パス／alt=代替文／width・height=レイアウトずれ防止／loading="lazy"=画面外画像の遅延読み込み

<a href="https://example.com" target="_blank" rel="noopener noreferrer">外部リンク</a>
// href=リンク先 URL（a=anchor、href=hyperlink reference）
// target="_blank" で新規タブ。その際は rel="noopener noreferrer" を必ず付けて
// タブ越しに親ページを操作される脆弱性（タブナビング）を防ぐ
ポイント
  • img や br のような単独タグは閉じタグを書かない。それ以外は基本「開いたら閉じる」が鉄則。
  • img には width/height を指定してレイアウトシフト（CLS）を防止。loading="lazy" でファーストビュー外の画像を遅延読込。
  • WebP / AVIF を切り替えるなら <picture> + <source> を使う。

HTH5a タグの擬似クラス

<a> タグはマウスの状態によって4 つの擬似クラスで色を切り替えられます。

擬似クラス 状態
a:link 未訪問のリンク
a:visited 訪問済みのリンク
a:hover カーソルが重なった時
a:active クリックしてから離すまで

記述例

a:link    { color: #0000FF; }
a:visited { color: #00FF00; }
a:hover   { color: #FFFF00; }
a:active  { color: #00FFFF; }

ライブサンプル

未訪問 訪問済み ホバー風 アクティブ風

注意

セレクタの a と :link の間は半角を空けずにくっつけるのが鉄則（a:link）。

重要：書く順序は LVHA を厳守

Link → Visited → Hover → Active の順で書かないと、後の指定が CSS のカスケードで打ち消されて効かなくなります（同じ詳細度の場合、後に書いたものが優先されるため、例えば :hover の後に :link を書くと :link が常に勝ってしまう）。さらにキーボード操作対応として :focus-visible を加えるとアクセシビリティが向上します。

HTH6構造タグ・テーブル

ページを意味あるブロックに分けるための、ひと回り大きい構造タグを学びます。

テーブル（表組み）

<table>
  <caption>表のタイトル</caption>
  <thead>
    <tr><th>見出し</th><th>見出し</th></tr>
  </thead>
  <tbody>
    <tr><td>セル</td><td>セル</td></tr>
  </tbody>
</table>

表全体を <table>、横 1 列を <tr>（table row）、見出しセルを <th>（table header cell）、データセルを <td>（table data cell）で囲みます。見出し行は <thead>、データ行は <tbody>、表の説明は <caption> で構造化するとアクセシビリティが向上します。

セクション系タグ

タグ 役割
<header> サイト上部のグローバルメニューなどに使う。<head> とは別物
<main> そのページの主題となる部分。1 ページに 1 つだけ置くのが現代の標準（スクリーンリーダーが本文ジャンプに使うため、アクセシビリティ・SEO 両面で強く推奨）
<nav> 主要なナビゲーション領域を表すセマンティックタグ。グローバルメニューなどを囲う
<article> 独立して成立するコンテンツ（ブログ記事・ニュース記事など）
<aside> 本文と直接関係のない補足情報（サイドバー・関連リンクなど）
<footer> 最下部。著作権表記やその他情報を入れる
<section> 見出しとその後の段落などを囲うひとかたまり。div より大きな意味を持たせる
<div> 何かを囲う汎用ブロック（block box）。意味は持たない
<span> 文章の一部だけにスタイルを当てる汎用インラインタグ（inline box）

典型的な骨組み

<body>
  <header></header>
  <main>
    <section><div></div></section>
    <section><div></div></section>
  </main>
  <footer></footer>
</body>

CSC1CSS の基本（セレクタ・プロパティ・値）

CSS は「どこに、何を、どう適用するか」を指定する言語です。 

p { color: red; }
// ↑セレクタ ↑プロパティ ↑値
用語 意味
セレクタ どこに適用するか（例：p 要素全部）
プロパティ名 何を適用するか（例：color）
プロパティ値 どのように適用するか（例：red）

上記の例は「すべての <p> 要素の文字色を赤にする」という意味になります。プロパティ名に応じて使える値が変わるので、辞書のように調べながら覚えていきます。

CSC2セレクタの基本

CSS には目的別に多くのセレクタがあります。よく使うものを 1 セクションでまとめて押さえます。

名前 書き方 意味
タイプセレクタ A {～} すべての A 要素に適用

CSC9box-sizing: border-box は絶対入れる

CSS の初期値では width / height に border と padding が含まれません。これを変更しないと、レイアウトが崩れる原因になります。 

// 現代の定番（擬似要素まで含めて全要素に当てる）
*, *::before, *::after { box-sizing: border-box; }
なぜ必須？

初期値（content-box）だと width: 200px に padding: 10px を足すと実サイズが 220px になり、グリッドが崩れます。border-box にしておけば width がそのまま外形サイズになり、コーディングしやすくなります。

* だけでは ::before / ::after 擬似要素には適用されないため、サイズ計算が content-box のまま残って思わぬズレを生みます。*, *::before, *::after までセットで指定するのが 2026 年の標準です。

$(function() {

});

意味は「HTML の読み込みが終わったら、中の処理を実行する」。HTML が読み込まれる前に処理を始めると要素が見つからずエラーになるため、この合図が必須です。

$ ってどういう意味？

$ は jQuery を短く書いたエイリアス（別名）です。 本来は jQuery() と書きますが、長いので $() と省略して使えるようになっています。 

// 下の 2 つは書き方が違うだけで同じ意味
jQuery('#main').hide();
$('#main').hide();

なぜ $ を使うのか？

  • 短くて書きやすい。
  • コードが読みやすい（$ が出てきたら「ここは jQuery だな」とすぐ分かる）。
注意

WordPress や他のライブラリと同時に使うと $ が別の意味で使われている場合があります。 そのときは省略せずに jQuery() と書く必要があります。

// 短い書き方
$(function() {

});

// 少し長いけど意味が同じ書き方
$(document).ready(function() {
  // 命令
});

どちらかを必ず使い、処理は必ず { } の中に書きます。

jQQ6セレクタ・メソッド・引数の詳細

命令の構文は「主語（セレクタ）＋動詞（メソッド）＋目的語（引数）」のかたちです。

セレクタ — 命令の主語

「どこの箇所に対して操作したい」と命令対象を指定するための部品です。 

$('要素名')

$('.クラス名')

2 つめのコードを書き換えてみよう

確認後はもとに戻しておきましょう：

メソッド — 命令の内容

命令の動詞にあたる部分。CSS を操作する css メソッド、HTML を操作する html メソッド、属性を操作する attr メソッドなどがあります。 

$('HTML 要素').メソッド('引数1', '引数2');
  • $() → 関数。jQuery オブジェクトを作る。
  • . → jQuery オブジェクトとメソッドをつなげる役割。
  • ; → 処理が終わったことを示す役割。

  • 'color' → 変更するプロパティ名（何を変えるか）。
  • '#ebc000' → プロパティの値（どう変えるか）。
  • カンマで区切って複数指定することも可能。

jQQ7複数のプロパティを一括変更する方法

jQuery practice
font-size: 50px ／ background-color: #ae5b9e ／ color: #ebc000 を一括適用した例

1 つずつ書いた場合

② JS を入力

  });
});
return false の意味

HTML 本来の機能を実行させないようにするための一行。 この場合はアンカー要素が本来持っているリンクの機能（ページ遷移）が実行されないようにするために必要です。 click を使うときは必ず入れると覚えておきましょう。

ビューアーの解説

① $('a').click(function(){ ... });

  • ページ内の <a> タグ（リンク）をクリックしたときに、{ } の中の処理を実行する。
  • 今回はサムネイル画像が <a> で囲まれている。

② $(this).attr('href')

  • this →「いまクリックされた a 要素自身」。
  • .attr('href') → その a 要素が持っているリンク先の URL を取り出す。
  • つまり「クリックしたサムネイルのリンク先 URL」を取得する。

③ $('figure img').attr('src', ~)

  • $('figure img') → <figure> 内の <img> 要素を指定（大きな画像が表示される場所）。
  • .attr('src', ~) → その <img> の src 属性を書き換える。
  • 意味：「大きな画像の表示先を、クリックしたサムネイルのリンク先に変更する」。

④ return false;

  • <a> タグはクリックすると本来「リンク先に飛ぶ」動作をする。
  • でも今回は「ページ遷移させずに、画像だけ切り替えたい」。
  • そのため本来のリンク動作をキャンセルしている。

GSG1GSAP の基本（ScrollTrigger まで）

GSAP（GreenSock Animation Platform）は、Web サイトのアニメーション演出で定番の JavaScript ライブラリです。 jQuery よりもアニメーション専門で、ScrollTrigger プラグインを組み合わせると「スクロール量に応じて動かす」「要素が画面に入ったら動かす」といった演出が短いコードで書けます。

1. 読み込み（CDN）

HTML の </body> 直前に CDN を読み込むだけで使えます。ScrollTrigger を使う場合は本体と一緒に読み込みます。 

<!-- GSAP 本体 -->
<script src="https://cdn.jsdelivr.net/npm/gsap@3/dist/gsap.min.js"></script>
<!-- ScrollTrigger プラグイン（スクロール連動を使うときだけ） -->
<script src="https://cdn.jsdelivr.net/npm/gsap@3/dist/ScrollTrigger.min.js"></script>
<script>
  gsap.registerPlugin(ScrollTrigger); // プラグイン登録（必須）
</script>

2. 基本構文（to / from / fromTo / set / timeline）

GSAP のアニメーションは「対象セレクタ」と「変化させる値」を渡すだけ。CSS のプロパティ名はキャメルケース（backgroundColor など）で書きます。

gsap.to() ― 現在の状態 → 指定した値へ動かす（一番よく使う） 

// .box を 1 秒かけて右に 200px 移動・回転
gsap.to('.box', {
  x: 200,
  rotation: 360,
  duration: 1,
  ease: 'power2.out'
});

gsap.from() ― 指定した値 → 現在の状態へ動かす（登場アニメーション向き） 

// 透明・下 40px の状態から、本来の位置へフェードイン
gsap.from('.fade-in', {
  opacity: 0,
  y: 40,
  duration: 0.8
});

gsap.fromTo() ― 開始値と終了値を両方明示

gsap.fromTo('.box',
  { opacity: 0, scale: 0.8 },   // from
  { opacity: 1, scale: 1, duration: 0.6 } // to
);

gsap.set() ― アニメさせず、値を即座にセット（初期化用） 

gsap.set('.box', { opacity: 0, y: 40 });

gsap.timeline() ― 複数のアニメを順番につなげる

// 見出し → リード文 → ボタン の順に登場
// const tl = gsap.timeline({ defaults: { duration: 0.6, ease: 'power2.out' } });
// tl.from('.h1', { y: 40, opacity: 0 })
//   .from('.lead', { y: 20, opacity: 0 }, '-=0.3') // 0.3 秒早く重ねる
//   .from('.btn',  { scale: 0.8, opacity: 0 });

3. ScrollTrigger の使い方（基本）

各 GSAP メソッドのオプションに scrollTrigger を渡すだけで、スクロールに連動します。 

gsap.from('.section-title', {
  opacity: 0,
  y: 60,
  duration: 0.8,
  scrollTrigger: {
    trigger: '.section-title', // 監視する要素
    start: 'top 80%',           // 要素の上端が画面の80%に来たら開始
    end: 'bottom 20%',         // 終了位置（scrub 時に使う）
    toggleActions: 'play none none reverse', // 入る/出る時の挙動
    // markers: true,  // 開発中はこれを入れると start/end が見える
  }
});

start の読み方: 'top 80%' = 「対象要素の上端」が「ビューポートの上から80%の位置」に来たら発火。 markers: true を入れると、画面に start/end のラインが表示されて確認しやすくなります。

4. ScrollTrigger で CSS クラスを付け替える

「スクロールしたらクラスを付け外しして、CSS 側でアニメーションさせる」やり方。CSS の transition や @keyframes と組み合わせて使えます。 

// .target が画面に入ったら is-active を付与、出たら外す
ScrollTrigger.create({
  trigger: '.target',
  start: 'top 75%',
  end: 'bottom 25%',
  toggleClass: { targets: '.target', className: 'is-active' },
  // once: true,  // 1 度だけ付与したいときは true
});

CSS 側はこんな感じで書きます。

.target { opacity: 0; transform: translateY(40px); transition: .6s ease; }
.target.is-active { opacity: 1; transform: none; }

JS 側はトリガーだけ担当、見た目は CSS に任せる方法です。デザイン担当が CSS だけ調整できるので、チーム制作で扱いやすい書き方です。

5. スクロール連動エフェクト（scrub / pin）

scrub を使うと、アニメーションの進行度がスクロール量に紐づき、巻き戻しもできる連動アニメになります。 

// スクロールに合わせて 0deg → 360deg 回転
gsap.to('.spin', {
  rotation: 360,
  ease: 'none',
  scrollTrigger: {
    trigger: '.spin',
    start: 'top 80%',
    end: 'bottom 20%',
    scrub: true,   // true = スクロールに同期 / 数値 = なめらかな追従（例: 0.5）
  }
});

pin を使うと、要素を画面に固定したままアニメーションを進められます（横スクロール演出やセクション固定に便利）。 

gsap.to('.panel', {
  xPercent: -100,         // 自身の幅ぶん左へ動かす
  ease: 'none',
  scrollTrigger: {
    trigger: '.panel-wrap',
    start: 'top top',
    end: '+=1500',        // 1500px ぶんスクロールするまで固定
    pin: true,
    scrub: true,
  }
});

パララックス（背景がゆっくり動く）も scrub と yPercent の組み合わせで簡単に作れます。 

gsap.to('.bg-image', {
  yPercent: -20,         // 上方向に少しだけ動かす
  ease: 'none',
  scrollTrigger: {
    trigger: '.bg-image',
    start: 'top bottom', // 画面に入った瞬間から
    end: 'bottom top',   // 画面から出るまで連動
    scrub: true,
  }
});
💡
使い分けの目安: 一度きりの登場アニメは toggleActions、 スクロール量と完全連動させたいなら scrub、 CSS 側に演出を任せたいなら toggleClass。
開発中は必ず markers: true を入れて start / end を目で確認しましょう。
